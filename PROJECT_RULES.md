# PROJECT_RULES.md

Regole e convenzioni di sviluppo per il progetto RoadTo100.

---

## Git Workflow

- Prima di ogni `commit` eseguire `git status` per verificare i file in staging.
- Non effettuare `push` senza una richiesta esplicita.
- Usare messaggi di `commit` descrittivi e leggibili.
- Non committare file temporanei, cache Python, `.pyc`, `__pycache__`, archivi `.zip`/`.7z` o altri file generati inutili.
